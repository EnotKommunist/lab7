**Задание 1**

**Задача 1**

**Текст задачи**

Для заданного файла возвратить true, если он содержит заданное число b, и false в
противном случае.

**Алгоритм решения**
Метод GenerateFileTask1 создаёт текстовый файл, в котором каждое случайное целое число записывается на отдельной строке. Для этого открывается поток записи StreamWriter, и в цикле for, выполняющемся count раз, генерируется случайное число в диапазоне от min до max с помощью random.Next, после чего оно немедленно выводится в файл вызовом writer.WriteLine. После завершения цикла поток автоматически закрывается благодаря конструкции using, и файл сохраняется на диске. Второй метод ContainsNumber проверяет, содержится ли в уже существующем файле заданное число b. Он построчно читает файл с помощью File.ReadLines, что не загружает весь файл в память. Для каждой прочитанной строки вызывается метод Trim, убирающий лишние пробелы, а затем int.TryParse пытается преобразовать строку в целое число. Если преобразование успешно и полученное число совпадает с искомым b, метод немедленно возвращает true. Если цикл завершается, а совпадение так и не найдено, возвращается false.

**Задание 2**

**Задача 1**

**Текст задачи**

Вычислить сумму элементов, которые кратны заданному числу k


**Алгоритм решения**

Метод GenerateFileTask2 предназначен для создания текстового файла, в котором на каждой строке располагается несколько целых чисел, разделённых пробелами. Количество строк задаётся параметром linesCount, а количество чисел в каждой строке — параметром numsPerLine. Открыв поток StreamWriter, программа выполняет внешний цикл, отвечающий за строки. На каждой итерации этого цикла создаётся пустой список numbers типа List<int>. Внутренний цикл выполняется numsPerLine раз, добавляя в список очередное случайное число из указанного диапазона. После завершения внутреннего цикла все числа списка объединяются в одну строку через пробел с помощью string.Join, и полученная строка записывается в файл вызовом writer.WriteLine. Следующий метод SumMultiplesOfK решает задачу вычисления суммы всех чисел из файла, которые кратны заданному k. Он открывает файл для построчного чтения, каждую строку разбивает на подстроки методом Split, используя пробел в качестве разделителя. Для каждой подстроки выполняется попытка преобразования в целое число. Если TryParse успешен и остаток от деления числа на k равен нулю, число добавляется к переменной sum. По окончании обработки всех строк и всех чисел возвращается накопленная сумма.**Тестирование**

**Задание 3**

**Задача 1**

**Текст задачи**

Переписать в другой файл строки, в которых нет цифр.

**Алгоритм решения**

Метод CopyLinesWithoutDigits копирует из исходного текстового файла в целевой только те строки, которые не содержат ни одной цифры. Сначала одновременно открываются StreamReader для чтения исходного файла и StreamWriter для записи в целевой. Затем в цикле while построчно читается исходный файл. Условие line = reader.ReadLine() != null означает, что на каждой итерации в переменную line считывается очередная строка, и как только будет достигнут конец файла, ReadLine вернёт null, что приведёт к выходу из цикла. Для каждой прочитанной строки проверяется наличие цифр с помощью line.Any(char.IsDigit). Метод Any является встроенным методом расширения, но в задании не было ограничений на его использование. Если строка не содержит ни одной цифры, она записывается в целевой файл. После обработки всех строк оба потока автоматически закрываются, и файлы сохраняются.

**Задание 4**

**Задача 1**

**Текст задачи**

Из исходного файла получить новый файл, исключив повторные вхождения чисел.

**Алгоритм решения**

Генерация бинарного файла производится в методе GenerateBinaryFileTask4. Открывается поток BinaryWriter, связанный с файлом, создаваемым через File.Open с режимом FileMode.Create. В цикле нужное количество раз генерируются случайные целые числа, каждое из которых записывается в файл методом writer.Write, сохраняющим число в бинарном виде как четыре байта. Метод RemoveDuplicatesBinary удаляет повторные вхождения чисел из бинарного файла. Он открывает исходный файл для чтения через BinaryReader и целевой файл для записи через BinaryWriter. Для отслеживания уже встреченных чисел используется множество HashSet<int> с именем seen. В цикле while, пока позиция чтения не достигла длины файла, считывается очередное целое число методом reader.ReadInt32. Если множество seen ещё не содержит это число, оно добавляется в seen и записывается в целевой файл. Если же число уже присутствует в seen, оно игнорируется, и цикл переходит к следующему числу. Таким образом, в результирующем файле остаются только уникальные числа в порядке их первого появления.

**Задание 5**

**Задача 1**

**Текст задачи**

Файл содержит сведения об игрушках: название игрушки, ее стоимость в рублях и возрастные
границы (например, игрушка может предназначаться для детей от двух до пяти лет). Получить
название игрушек, цена которых не превышает k руб. и которые подходят детям 5 лет.


**Алгоритм решения**

Метод GenerateXmlFileTask5 создаёт XML-файл со сведениями об игрушках. Сначала инициализируется пустой список объектов класса Toy. Задаётся массив строк names с возможными названиями игрушек. В цикле, выполняющемся count раз, создаётся новый объект Toy, его свойствам присваиваются случайные значения: название формируется из случайного элемента массива names с добавлением порядкового номера, цена генерируется в диапазоне от 50 до 1500 рублей, минимальный возраст — от 1 до 5 лет, максимальный возраст — от 5 до 12 лет. Созданный объект добавляется в список toys. После завершения цикла создаётся объект XmlSerializer для типа List<Toy>, которому передаётся корневой атрибут с именем "Toys". Затем открывается файловый поток FileStream для создания файла, и сериализатор записывает в него список игрушек в формате XML. Второй метод GetAffordableToysForAge5 решает задачу поиска подходящих игрушек. Он принимает путь к XML-файлу и максимальную цену k. Создаётся аналогичный XmlSerializer, открывается файловый поток для чтения, и вызывается метод Deserialize, который восстанавливает список игрушек из XML. Результат приводится к типу List<Toy>. Затем в цикле foreach перебираются все игрушки, и для каждой проверяются три условия: цена не превышает k, минимальный возраст меньше или равен пяти годам, максимальный возраст больше или равен пяти годам. Если все условия выполняются, название игрушки добавляется в результирующий список. По окончании обработки список названий возвращается. Класс Toy содержит приватные поля name, price, minAge, maxAge и публичные свойства для доступа к ним, что соответствует стандартной инкапсуляции данных. Все методы в классе Class1 объявлены статическими, что позволяет вызывать их без создания экземпляра класса.

**Задание 6**

**Задача 1**

**Текст задачи**

Составить программу, которая удаляет из списка L все элементы с указанным значением.

**Алгоритм решения**

Метод RemoveAllElements решает задачу удаления из списка всех элементов с заданным значением. Это обобщённый метод, способный работать со списками любого типа T. Внутри метода создаётся новый пустой список result, который будет наполняться только теми элементами, которые не равны удаляемому значению. Далее организуется цикл foreach, перебирающий все элементы исходного списка. Для каждого элемента вызывается метод Equals, сравнивающий его с переданным значением value. Если результат сравнения отрицательный, то есть элемент не равен удаляемому значению, он добавляется в результирующий список. После завершения перебора всех элементов метод возвращает сформированный список, содержащий все элементы исходного, кроме удаляемых. Важно отметить, что исходный список остаётся неизменным, а создаётся его модифицированная копия.

**Задание 7**

**Задача 1**

**Текст задачи**

В списке L переставить в обратном порядке все элементы между первым и последним
вхождениями элемента E, если E входит в L не менее двух раз.

**Алгоритм решения**

Метод ReverseBetweenFirstAndLast выполняет перестановку в обратном порядке элементов связного списка LinkedList, расположенных между первым и последним вхождением заданного элемента. В начале проверяется, что список содержит хотя бы два элемента, иначе операция не имеет смысла и метод завершается. Затем осуществляется поиск первого вхождения: переменной firstNode присваивается ссылка на первый узел списка, после чего в цикле while происходит переход к следующим узлам до тех пор, пока не будет найден узел, значение которого равно искомому элементу. Аналогично ищется последнее вхождение, но поиск начинается с конца списка с использованием ссылки lastNode и свойства Previous для движения к началу. Если элемент не найден или первое и последнее вхождения совпадают, то есть элемент встречается менее двух раз, метод завершается. Если оба узла найдены и они различны, создаётся временный список middleElements типа List. В цикле, начиная с узла, следующего за firstNode, и до узла, предшествующего lastNode, все значения элементов добавляются в этот список. Далее выполняется переворот полученного списка путём обмена симметричных элементов: в цикле от нуля до половины длины списка элементы с индексами i и длина минус один минус i меняются местами через временную переменную. После переворота временного списка его элементы записываются обратно в связный список: переменная current снова устанавливается на узел после firstNode, и в цикле каждому узлу присваивается соответствующее значение из перевёрнутого списка middleElements. Таким образом, часть связного списка между первым и последним вхождением элемента оказывается перевёрнутой, а сам элемент и всё, что находится за его пределами, остаётся на своих местах.

**Задание 8**

**Задача 1**

**Текст задачи**

Есть перечень фильмов. Определить для каждого фильма, какие из них посмотрели все n
зрителей, какие — некоторые из зрителей, и какие — никто из зрителей.

**Алгоритм решения**

Метод AnalyzeMovieViews анализирует просмотры фильмов группой зрителей и распределяет фильмы по трём категориям: просмотренные всеми зрителями, просмотренные некоторыми и не просмотренные никем. Метод принимает список множеств viewersMovies, где каждое множество содержит названия фильмов, просмотренных отдельным зрителем, а также полный список всех фильмов allMovies. Через выходные параметры с ключевым словом out метод возвращает три множества. Внутри метода определяется общее количество зрителей n, инициализируются три пустых множества для результатов. Затем организуется внешний цикл по всем фильмам из allMovies. Для каждого фильма запускается внутренний цикл, который перебирает всех зрителей и с помощью метода Contains проверяет, содержится ли текущий фильм в множестве просмотренных фильмов данного зрителя. Если содержится, увеличивается счётчик count. После обработки всех зрителей для текущего фильма значение счётчика анализируется: если count равно общему числу зрителей n, фильм добавляется в множество watchedByAll; если count больше нуля, но меньше n, фильм добавляется в watchedBySome; если count равен нулю, фильм добавляется в watchedByNone. По завершении внешнего цикла все три результирующих множества заполнены и доступны вызывающему коду через выходные параметры.

**Задание 9**

**Задача 1**

**Текст задачи**

Файл содержит текст на русском языке. Напечатать в алфавитном порядке все звонкие
согласные буквы, которые входят более чем в одно слово.


**Алгоритм решения**

Метод FindFrequentVoicedConsonants находит в текстовом файле все звонкие согласные буквы русского алфавита, которые встречаются более чем в одном слове, и возвращает их в алфавитном порядке. В начале метода создаётся множество voicedConsonants, содержащее все звонкие согласные буквы в обоих регистрах. Затем инициализируется словарь letterCount, где ключами выступают символы этих букв, а значениями — счётчики, изначально равные нулю. Для каждой заглавной буквы дополнительно добавляется ключ со строчной версией. Далее весь текст из файла считывается в строку с использованием кодировки UTF-8, после чего вызывается вспомогательный метод SplitIntoWords, разбивающий текст на массив отдельных слов. Для каждого слова создаётся множество lettersInWord, предназначенное для хранения уникальных звонких согласных, встретившихся в этом слове. В цикле по символам слова каждый символ приводится к нижнему регистру, и если он присутствует в множестве voicedConsonants, добавляется в lettersInWord. После обработки всех символов слова для каждой буквы из lettersInWord увеличивается соответствующий счётчик в словаре letterCount. Когда все слова обработаны, формируется результирующий список result: в цикле по парам ключ-значение словаря letterCount проверяется, превышает ли значение счётчика единицу. Если да, ключ добавляется в result. Наконец, полученный список сортируется по алфавиту вызовом вспомогательного метода SortList, реализующего пузырьковую сортировку, и возвращается. Вспомогательный метод SplitIntoWords просматривает текст посимвольно, накапливая буквы в объекте StringBuilder. Как только встречается символ, не являющийся буквой, накопленное слово добавляется в список, а построитель очищается. Поддерживаются русские буквы ё и Ё. В конце, если после обработки всего текста в построителе осталось слово, оно также добавляется.
